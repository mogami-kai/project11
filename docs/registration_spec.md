# Registration Spec (v5)

## 0. Purpose
LINE userId と本人情報を確実に紐づけ、提出ログを本人単位で一貫運用するための「本人特定・連絡・緊急対応」情報を取得する。

---

## 1. Required Fields（全員必須）

| フィールド | 内容 |
|-----------|------|
| 氏名 | 漢字・姓と名にスペース |
| フリガナ | 全角カナ・姓と名にスペース |
| 生年月日 | 日付 |
| 最寄駅 | 曖昧駅は県名等で補足 |
| 携帯番号 | 正規化して保存 |
| 緊急連絡先 | 関係 + 電話 |
| 郵便番号 | 正規化して保存 |
| 住所 | フル住所 |

---

## 2. Validation Rules

- **氏名：** `姓 名`（スペース必須）
- **フリガナ：** `セイ メイ`（全角カナ + スペース必須）
- **生年月日：** 日付以外不可
- **携帯番号：** 数字/ハイフン許可 → 保存時は正規化（例: `09012345678`）
- **郵便番号：** 数字/ハイフン許可 → 保存時は正規化（例: `1234567`）

---

## 3. Gating Rules
登録が完了していない場合：
- リッチメニュー（提出系）非表示 or 押下不可
- 提出系APIは `E_REGISTER_REQUIRED` で拒否
- 登録画面への誘導メッセージを返す

---

## 4. Master Fields（運用強化）

| フィールド | 用途 |
|-----------|------|
| `isActive` | 在籍/無効（退職・休職等の論理削除） |
| `lineFollowStatus` | follow/unfollow（配信対象絞り込み） |
| `updatedAt` / `updatedBy` | 本人/管理者/システムの更新追跡 |
| `dataHash` | 改ざん検知用 |

---

## 5. Privacy
- 本人情報は Staff Master が正本
- 提出ログには原則 lineUserId のみ保持し、必要に応じてスナップショットを付与（v5では任意）
